# SSH Management

## Objective

The objective of this phase is to replace insecure Telnet access with encrypted SSH remote management.

This implementation improves the security of network device administration by allowing authenticated and encrypted remote access.

---

## Why SSH?

Telnet transmits usernames and passwords in plain text.

SSH encrypts all management traffic, protecting administrator credentials and configuration sessions.

---

## Devices

- Cisco Router
- Cisco Catalyst Switch

---

## Features

- Local User Authentication
- SSH Version 2
- RSA Key Generation
- Encrypted Passwords
- Login Banner
- Secure VTY Access
